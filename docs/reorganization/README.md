# Project Record Reorganization

本目录记录 2026-07-12 对 OmniFocus MCP 项目本地材料的只读盘点、实际方案和保守整理结果。

## 阅读顺序

1. [Source Inventory](./00-source-inventory.md)：20 份项目 Markdown 的逐文件分类与证据。
2. [Topic Matrix](./01-topic-matrix.md)：关键主题的权威来源、历史材料和下一步依赖。
3. [Conflicts and Superseded Material](./02-conflicts-and-superseded-material.md)：冲突、时间线变化和被取代范围。
4. [Actual Reorganization Plan](./03-actual-reorganization-plan.md)：基于真实扫描结果形成的 KEEP/INDEX/SYNTHESIZE 方案。

## 结果摘要

事实：仓库内未发现原始 ChatGPT/Codex 对话导出；主要材料是工程日志、Accepted ADR、架构/集成/运维文档。所有原文均原位保留，本轮未移动、重命名、归档或删除任何原件。

判断：最需要解决的是权威入口和时间线不清，而不是目录数量。

建议：从 [项目文档导航](../README.md)、[当前状态](../PROJECT_STATUS.md) 和 [来源映射](../SOURCE_MAP.md) 进入；需要工程历史时再阅读原始日志和[演进总结](../history/evolution-summaries/)。

## 范围限制

本次只能检查当前工作区和用户明确提供的外部指令文件。未导出到本地的 ChatGPT 云端历史、其他 Project 对话和未挂载笔记不在可访问范围内，因此这里不是“全部聊天历史”的审计。
